<p>In <a href="/wiki/Computing" title="Computing">computing</a>, <b><tt>ls</tt></b> is a <a href="/wiki/Unix_command" class="mw-redirect" title="Unix command">command</a> to <i>list</i> files in <a href="/wiki/Unix" title="Unix">Unix</a> and <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> operating systems. <tt>ls</tt> is specified by <a href="/wiki/POSIX" title="POSIX">POSIX</a> and the <a href="/wiki/Single_UNIX_Specification" title="Single UNIX Specification">Single UNIX Specification</a>. When invoked without any arguments, ls lists the files in the <a href="/wiki/Working_directory" title="Working directory">current working directory</a>.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ls&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>An <tt>ls</tt> utility appeared in the original version of <a href="/wiki/AT%26T_Corporation" title="AT&amp;T Corporation">AT&amp;T</a> <a href="/wiki/Unix" title="Unix">UNIX</a>. Today, two popular versions of <tt>ls</tt> are the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a>'s (part of the <a href="/wiki/GNU" title="GNU">GNU</a> <a href="/wiki/GNU_Core_Utilities" title="GNU Core Utilities">coreutils</a> package) and the one released by various <a href="/wiki/Berkeley_Software_Distribution" title="Berkeley Software Distribution">BSD</a> variants, such as <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a>, <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a>, <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a>, and <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple's</a> <a href="/wiki/Darwin_(operating_system)" title="Darwin (operating system)">Darwin</a>. Both are <a href="/wiki/Free_software" title="Free software">free software</a> and <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open source</a>. The name 'ls' comes from a similar command in <a href="/wiki/Multics" title="Multics">Multics</a> also named 'ls', short for the word "list".<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup><sup class="noprint Inline-Template noprint Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:NOTRS" class="mw-redirect" title="Wikipedia:NOTRS"><span title="This claim needs references to better sources. (October 2015)">better&#160;source&#160;needed</span></a></i>]</sup></p>
<h2><span class="mw-headline" id="Behavior">Behavior</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ls&amp;action=edit&amp;section=2" title="Edit section: Behavior">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Unix" title="Unix">Unix</a> and <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> operating systems maintain the idea of a <i><a href="/wiki/Working_directory" title="Working directory">current working directory</a></i>, that is, where one is currently positioned in the hierarchy of <a href="/wiki/Directory_(file_systems)" class="mw-redirect" title="Directory (file systems)">directories</a>.</p>
<p>When invoked without any arguments, <tt>ls</tt> lists the files in the current working directory. A directory that is not the current working directory can be specified and <tt>ls</tt> will list the files there. The user also may specify any list of files and directories. In this case, all files and all contents of specified directories will be listed.</p>
<p>Files whose names start with "." are not listed, unless the <tt>-a</tt> flag is specified, the <tt>-A</tt> flag is specified, or the files are specified explicitly.</p>
<p>Without options, <tt>ls</tt> displays files in a bare format. This bare format however makes it difficult to establish the type, permissions, and size of the files. The most common options to reveal this information or change the list of files are:</p>
<ul>
<li><tt>-l</tt> long format, displaying <a href="/wiki/Unix_file_types" title="Unix file types">Unix file types</a>, permissions, number of <a href="/wiki/Hard_link" title="Hard link">hard links</a>, owner, group, size, last-modified date and filename</li>
<li><tt>-f</tt> do not sort. Useful for directories containing large numbers of files.</li>
<li><tt>-F</tt> appends a character revealing the nature of a file, for example, <tt>*</tt> for an executable, or <tt>/</tt> for a directory. Regular files have no suffix.</li>
<li><tt>-a</tt> lists all files in the given directory, including those whose names start with "." (which are <a href="/wiki/Hidden_file_and_hidden_directory" title="Hidden file and hidden directory">hidden files</a> in Unix). By default, these files are excluded from the list.</li>
<li><tt>-R</tt> recursively lists subdirectories. The command <tt>ls -R /</tt> would therefore list all files.</li>
<li><tt>-d</tt> shows information about a symbolic link or directory, rather than about the link's target or listing the contents of a directory.</li>
<li><tt>-t</tt> sort the list of files by modification time.</li>
<li><tt>-h</tt> print sizes in human readable format. (e.g., 1K, 234M, 2G, etc.) This option is not part of the POSIX standard, although implemented in several systems, e.g., GNU coreutils in 1997,<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> FreeBSD 4.5 in 2002,<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> and Solaris 9 in 2002.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></li>
</ul>
<p>In some environments, providing the option <tt>--color</tt> (for <a href="/wiki/GNU" title="GNU">GNU</a> <tt>ls</tt>) or <tt>-G</tt> (<a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> <tt>ls</tt>) causes <tt>ls</tt> to highlight different types of files with different colors, instead of with characters as <tt>-F</tt> would. To determine what color to use for a file, GNU <tt>ls</tt> checks the Unix file type, the file permissions, and the file extension, while FreeBSD <tt>ls</tt> checks only the Unix file type and file permissions. On the other hand, the FreeBSD implementation uses the termcap database<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup> while the GNU program uses its own database which does not necessarily match the termcap data on a given system.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup></p>
<p>When the option to use color to indicate file types is selected, the output might look like:</p>
<pre>
<strong><span style="background:#000000; color:#c0c0c0"> brw-r--r--    1 unixguy staff 64,  64 Jan 27 05:52 <span style="color:yellow">block         </span>
 crw-r--r--    1 unixguy staff 64, 255 Jan 26 13:57 <span style="color:#ffff00">character     </span>
 -rw-r--r--    1 unixguy staff     290 Jan 26 14:08 <span style="color:#ff00ff">compressed.gz </span>
 -rw-r--r--    1 unixguy staff  331836 Jan 26 14:06 <span style="color:#ff00ff">data.ppm      </span>
 drwxrwxr-x    2 unixguy staff      48 Jan 26 11:28 <span style="color:#0000ff">directory     </span>
 -rwxrwxr-x    1 unixguy staff      29 Jan 26 14:03 <span style="color:#00ff00">executable    </span>
 prw-r--r--    1 unixguy staff       0 Jan 26 11:50 <span style="color:#ffffaa">fifo          </span>
 srw-rw-rw-    1 unixguy staff       0 Jan 26 12:00 <span style="color:#ff00ff">socket        </span>
 lrwxrwxrwx    1 unixguy staff       3 Jan 26 11:44 <span style="color:#00ffaa">link</span> -&gt; <span style="color:blue">dir   </span>
 -rw-rw----    1 unixguy staff     217 Jan 26 14:08 regularfile   </span></strong>
</pre>
<p><tt>ls</tt> has a large number of other options (see: <tt><a href="/wiki/Unix_manual" class="mw-redirect" title="Unix manual">man</a> ls</tt>). It is a frequently used tool that provides an interface to the file system via the <a href="/wiki/Command_line" class="mw-redirect" title="Command line">command line</a>.</p>
<h2><span class="mw-headline" id="Sample_usage">Sample usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ls&amp;action=edit&amp;section=3" title="Edit section: Sample usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The following example demonstrates the output of the <tt>ls</tt> command given two different arguments (pwd is a command that shows the present working directory, or in other words, the folder you are currently in):</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">$</span> <span class="nb">pwd</span>
<span class="go">/home/fred</span>
<span class="gp">$</span> ls -l
<span class="go">drwxr--r--   1 fred  editors   4096  drafts</span>
<span class="go">-rw-r--r--   1 fred  editors  30405  edition-32</span>
<span class="go">-r-xr-xr-x   1 fred  fred      8460  edit</span>
<span class="gp">$</span> ls -F
<span class="go">drafts/</span>
<span class="go">edition-32</span>
<span class="go">edit*</span>
</pre></div>
<p>In this example, the user <tt>fred</tt> has a directory named <tt>drafts</tt>, a regular file called <tt>edition-32</tt>, and an executable named <tt>edit</tt> in his home directory. <tt>ls</tt> uses <a href="/wiki/File_system_permissions#Notation_of_traditional_Unix_permissions" title="File system permissions">Unix file permission notation</a> to indicate which users or groups are allowed to access each file or directory.</p>
<pre>
drwxr--r--   1 fred  editors   4096  Mar 1  2007 drafts
</pre>
<p>This means that the letters behind the file descriptor (d), which indicates a folder or 'directory', list three characters to indicate permissions for the owner of 'drafts' (rwx), then the group to which the file belongs (r--), and the rights of others (r--).</p>
<p>'drafts' is a directory (d), the owner of which has the right to read (r) write (w) and execute (x): rwx, group members have (r--), meaning read only, and others have (r--), meaning read only access. See <a href="/wiki/File_system_permissions#Permission_notation" title="File system permissions">Unix file permission notation</a> for a more detailed description.</p>
